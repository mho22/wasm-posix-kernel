# Wasm Dependency Management V2 — Chunk E (release tooling)

> **For Claude:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development`
> to execute this plan task-by-task.

**Goal:** Produce and consume `binaries-abi-v<N>/` archive bundles for
libraries + programs across `wasm32` and `wasm64`, with the existing
remote-fetch verification chain (sha + target_arch + abi_versions +
cache_key_sha) closing the loop end-to-end.

**Architecture:** Three new xtask subcommands stack on top of the
machinery Chunks A–D shipped:

  1. **`archive_stage`** packages a single resolved cache entry as
     `<staging>/<archive_name>.tar.zst` with an injected
     `[compatibility]` block in the embedded `manifest.toml`. This is
     the closing half of decision 14 — the consumer remote-fetch
     already verifies the four compatibility axes; until now there
     was no producer to test it against. Source-kind has no archive
     (decision 6).

  2. **`stage_release`** walks `Registry::walk_all()`, filters to
     `Library`/`Program` manifests, fans out across `{wasm32, wasm64}`,
     calls `ensure_built` to populate the cache, and invokes
     `archive_stage` on each. Output is a flat
     `binaries-abi-v<N>/{libs,programs}/*.tar.zst` tree plus a
     `manifest.json` regenerated by an extended `build_manifest`.

  3. **`install_release`** reads a release `manifest.json`, downloads
     each archive via `file://` or `https://`, and routes by entry
     kind: lib entries flow through the existing
     `remote_fetch::fetch_and_install` path into the per-arch cache;
     program entries also flow through `fetch_and_install` (so the
     compatibility chain is verified) and additionally have their
     wasm artifacts copied into `local-binaries/programs/<name>/` so
     `host/src/binary-resolver.ts` finds them without a cache lookup.

The two existing shell scripts (`scripts/stage-release.sh`,
`scripts/fetch-binaries.sh`) become thin wrappers around the new
xtask subcommands. `scripts/publish-release.sh` only needs minor
touch-ups to the asset-list discovery to handle the new
`{libs,programs}/` subdirectory layout. The release-side `manifest.json`
schema gains a parallel set of fields per entry — `archive_name`,
`archive_sha256`, plus a flat `compatibility` block carrying
`target_arch` / `abi_versions` / `cache_key_sha` so consumers can
reject stale archives without unpacking.

**Tech Stack:** Rust (xtask — `tar`, `zstd`, `sha2`, `serde_json`,
`toml`, `wasm_posix_kernel_shared::ABI_VERSION`); bash (release
scripts); TypeScript + Vitest (round-trip test); GitHub Releases (out
of scope for the round-trip test, exercised manually by Chunk F's
`binaries-abi-v<N>` cut).

## Acceptance criteria

### Schema (E.1)

- `abi/manifest.schema.json` accepts both V1-vintage entries (kind
  `kernel`/`userspace`/`program`/`vfs-image`, name carries
  short-sha hex, no compatibility block) and V2 lib/program archive
  entries.
- V2 entries set `kind: "library"` or `kind: "program"`, name format
  `<name>-<version>-rev<N>-<arch>-<short_sha>.tar.zst`, and carry an
  embedded `compatibility` object with `target_arch`,
  `abi_versions: [<u32>, ...]`, and `cache_key_sha` (64-char lower
  hex) so a fetcher can reject mismatched archives without unpacking.
- A `Compatibility` data check rejects malformed archive entries
  (missing or empty `abi_versions`, non-hex `cache_key_sha`).
- All fields are `additionalProperties: false`-compatible — adding
  unknown fields is a hard error.

### archive_stage (E.2)

- New module `xtask/src/archive_stage.rs` packages a resolved cache
  entry into a `.tar.zst` archive at a caller-specified output path.
- Tar layout matches what `remote_fetch::flatten_archive_layout`
  expects: top-level `manifest.toml` plus `artifacts/...` subtree.
- Embedded `manifest.toml` is the source `deps.toml` content with an
  injected `[compatibility]` block populated as
  `target_arch = <arch>`, `abi_versions = [<current_abi>]`,
  `cache_key_sha = <hex>`, `build_timestamp = <RFC3339 UTC>`,
  `build_host = <darwin-arm64 | linux-x86_64 | …>`.
- Round-trip test: pack → unpack via `remote_fetch::fetch_and_install`
  → all four verification steps pass (sha, target_arch,
  abi_versions, cache_key_sha) and the canonical cache layout is
  reproduced bit-identical to a direct cache build.
- Source-kind manifests reject at archive_stage entry — sources have
  no archive and no compatibility block.

### build_manifest (E.3)

- `xtask::build_manifest` learns to walk
  `Registry::walk_all()` filtered to `Library`/`Program` and emit a
  V2 entry per (manifest, arch) pair.
- Entry name follows the staged archive filename
  (`<name>-<version>-rev<N>-<arch>-<short_sha>.tar.zst`).
- Each entry's `compatibility.cache_key_sha` matches the resolver's
  locally computed sha for that (manifest, arch) — i.e. what
  `remote_fetch` will verify against on fetch.
- Existing V1-vintage `kernel`/`userspace`/`vfs-image` entry walking
  continues to work; the V2 path is additive.
- `--abi` flag is honored (defaults to current `ABI_VERSION`) so the
  generator can be exercised in tests without mutating the kernel.

### stage_release subcommand (E.4)

- New `cargo xtask stage-release --staging <dir> [--abi <N>]
  [--arch wasm32|wasm64]...` populates `<dir>/{libs,programs}/`
  with `.tar.zst` archives and writes `<dir>/manifest.json`.
- Default arch set: `[wasm32, wasm64]`. `--arch` can be repeated to
  restrict.
- Deterministic: re-running without rebuilding produces byte-identical
  archives (modulo `build_timestamp` — the test path supports
  `WASM_POSIX_BUILD_TIMESTAMP` override for stable golden output).
- Non-fatal arch skip: if a manifest's build script fails for one
  arch (e.g. wasm64 not yet supported by a particular library), the
  subcommand logs the failure and continues. Exit code is non-zero
  iff at least one manifest fails for *every* requested arch.

### install_release subcommand (E.5)

- New `cargo xtask install-release --manifest <path>
  [--archive-base <url-or-dir>]` reads the manifest, fetches each
  entry, and routes by kind.
- Lib entries: dispatched to `remote_fetch::fetch_and_install` →
  installed into `<cache>/libs/<canonical>/`.
- Program entries: same `remote_fetch::fetch_and_install` flow into
  `<cache>/programs/<canonical>/`, plus a copy step that mirrors
  declared `[[outputs]]` wasms into
  `local-binaries/programs/<output_name>.wasm` (single-output) or
  `local-binaries/programs/<program_name>/<output_name>.wasm`
  (multi-output, mirroring the existing
  `scripts/build-programs.sh` layout).
- `--archive-base` overrides the URL prefix in `archive_name`.
  Default is to use the URL embedded in `manifest.json` (when
  publish-time URLs are encoded); when only `archive_name` is
  present, the caller MUST pass `--archive-base file:///…/binaries-abi-v<N>`
  or an `https://github.com/…/releases/download/binaries-abi-v<N>`
  URL.
- Sha verification is enforced on every byte that lands on disk; an
  archive whose top-level sha mismatches `archive_sha256` is
  rejected. A subsequent compatibility-axis mismatch (target_arch /
  abi_versions / cache_key_sha) is treated as a hard install error
  rather than the resolver's "fall through to source build" semantics
  — the user invoked the install path explicitly.

### Shell wrappers (E.6 + E.7)

- `scripts/stage-release.sh` becomes a thin wrapper around
  `cargo xtask stage-release`. The legacy V1 `bundle-program` path
  for `kernel`/`userspace`/test programs is preserved (those entries
  do not have `deps.toml` registries) but moves to its own helper
  function — V2 lib/program staging is the dominant code path.
- `scripts/fetch-binaries.sh` invokes `cargo xtask install-release`
  with the manifest pinned in `binaries.lock`; the existing
  symlink-into-`binaries/` placement code that supports the V1
  `kernel.wasm`/`userspace.wasm`/`vfs/*.vfs.zst` artifacts stays.
- `scripts/publish-release.sh` learns to upload from the
  `{libs,programs}/` subdirectory layout — currently it does
  `find -maxdepth 1 -type f` and would miss the subdirs.

### Round-trip test (E.8)

- New `host/test/release-roundtrip.test.ts` (vitest, integration):
  - Picks a small fixture set (`zlib` + 1 program, e.g. `bc`).
  - Runs `cargo xtask stage-release --staging <tmp>` (host triple).
  - Hand-edits the generated `manifest.json`'s `archive_url` (or
    passes `--archive-base file://…`) to point at the staging dir.
  - Runs `cargo xtask install-release` against an isolated
    `XDG_CACHE_HOME` and `WASM_POSIX_LOCAL_BINARIES_DIR`.
  - Asserts the canonical cache directories exist with expected
    files, `local-binaries/programs/bc.wasm` exists, and
    `cargo xtask build-deps resolve zlib --arch wasm32` returns the
    fetched cache path without rebuilding (instrumentation: the
    fixture build script writes a marker so the test can prove the
    script did not run).
- The test gracefully skips on platforms where the host build of the
  fixtures is not available, mirroring the existing `php-bundle`
  skip logic.

### Documentation (E.9)

- `docs/dependency-management.md`: append a "Release archives"

- `docs/binary-releases.md`: replace the V1 zip-based program layout

## Caveats inherited from A–D

These already burned us; they shape every subagent prompt.

- **xtask requires `--target aarch64-apple-darwin`** (or matching
  host triple) due to ureq TLS deps on macOS. Documented in
  `xtask/README.md`.
- **libc-test has one pre-existing FAIL** (regression/daemon-failure)
  on the chunk-D.8 base; do not flag it as a new regression.
- **Pre-existing dirty `examples/libs/{curl,libcurl,wget,git,file,bc,
  nano,cpython,php,redis,mariadb}-src/*` files** in the worktree are
  build byproducts; never `git add` them.
- **`package-lock.json` may show a worktree-name diff**; not yours
  to commit.
- **Sortix `signal/ppoll-block-sleep-write-raise`** is documented as
  parallel-flaky; re-running in isolation usually passes. Do not
  treat a single failure under load as a regression.
- **SDK is worktree-local** (no `npm link` required). Source
  `sdk/activate.sh` near the top of any new build script — pattern
  is shipped on this branch.
- **`cmd_resolve` subprocess stdio** (PR #355): the resolver's
  build-script stdout goes to *stderr*, so
  `PREFIX="$(cargo run -- build-deps resolve <name>)"` works even on
  a cache miss. Don't fall back to the V1 "warm the cache first"
  pattern.
- **Compatibility-block parser** is already strict: `abi_versions`
  must be non-empty, `cache_key_sha` must be 64-char lower hex. Use
  the existing `validate_compatibility` codepath rather than
  reinventing it.
- **Always pass `model: "opus"` on Agent calls.** User preference;
  Sonnet/Haiku are off-limits.

**Background:** This module owns the producer side of decision 14.
It receives a resolved cache entry path and produces a `.tar.zst`
archive with:

```
archive root/
├── manifest.toml         # source deps.toml + injected [compatibility]
└── artifacts/
    ├── (lib/, include/, lib/pkgconfig/, ...)        for kind=library
    └── (*.wasm at top of artifacts/, one per [[outputs]])  for kind=program
```

`remote_fetch::flatten_archive_layout` (already shipped on this
branch) hoists `artifacts/*` to the cache-root layout on install,
so we MUST NOT include the cache's `*.wasm`-at-root layout directly
— the producer always wraps in `artifacts/`.

```rust
    // Build a fake "cache entry" with the canonical lib layout.
    let cache_entry = tempdir("e2-cache-entry");
    fs::create_dir_all(cache_entry.join("lib")).unwrap();
    fs::write(cache_entry.join("lib/libZ.a"), b"\x00\x01").unwrap();
    fs::create_dir_all(cache_entry.join("include")).unwrap();
    fs::write(cache_entry.join("include/zlib.h"), b"#pragma once\n").unwrap();

    // Construct a synthetic manifest matching the entry.
    let registry_root = tempdir("e2-registry");
    write_lib(&registry_root, "zlib", "1.0.0", &[],
        "true",
        "[outputs]\nlibs = [\"lib/libZ.a\"]\nheaders = [\"include/zlib.h\"]\n");
    let m = load_manifest(&registry_root, "zlib");

    // Stage.
    let staging = tempdir("e2-stage");
    let archive_path = staging.join("zlib-1.0.0-rev1-wasm32-deadbeef.tar.zst");
    let cache_key_sha = "00".repeat(32);  // arbitrary hex
    stage_archive_with_options(&m, TargetArch::Wasm32, 4,
        &cache_entry, &archive_path, &StageOptions {
            cache_key_sha: cache_key_sha.clone(),
            build_timestamp: "2026-04-26T00:00:00Z".into(),
            build_host: "darwin-arm64".into(),

    assert!(archive_path.is_file());

    // Consume via remote_fetch.
    let archive_bytes = fs::read(&archive_path).unwrap();
    let archive_url = format!("file://{}", archive_path.display());
    let archive_sha = sha256_hex(&archive_bytes);
    let cache_root = tempdir("e2-cache-root");

        TargetArch::Wasm32, 4, &cache_key_sha).unwrap();

    assert!(!canonical.join("manifest.toml").exists());
```

```rust
//! Producer side of the V2 binary release pipeline.
//!
//! Packages a resolved cache entry (the lib/program install tree
//! the resolver produced under `<cache>/<kind>/<name>-...`) into a
//! `.tar.zst` archive that the consumer-side
//! [`remote_fetch::fetch_and_install`] can verify and unpack.
//!
//! Decision 14 in
//! `docs/plans/2026-04-22-deps-management-v2-design.md`: the archive
//! carries a single `manifest.toml` (source `deps.toml` + injected
//! `[compatibility]` block) plus an `artifacts/` subtree holding the
//! built files. `flatten_archive_layout` on the consumer side hoists
//! `artifacts/*` to the cache-root layout post-extract.

use std::fs;
use std::io::Write;
use std::path::{Path, PathBuf};

use crate::deps_manifest::{Compatibility, DepsManifest, ManifestKind, TargetArch};

/// Caller-supplied build provenance + the locally-computed cache-key
/// sha. We don't recompute the sha here so the caller (`stage_release`
/// or a hand-driven build) can stay the source of truth on what the
/// consumer is required to match against.
pub struct StageOptions {
    pub cache_key_sha: String,        // 64 lowercase hex
    pub build_timestamp: String,      // RFC3339 UTC, e.g. "2026-04-26T10:00:00Z"
    pub build_host: String,           // e.g. "darwin-arm64", "linux-x86_64"
}

/// Pack the resolved cache entry at `cache_dir` into a `.tar.zst`
/// archive at `archive_path`. Errors leave `archive_path` absent;
/// success guarantees the archive is on disk and `unpack`-able.
///
/// The archive contents:
///   * `manifest.toml`   — `source_manifest_text(target)` + injected
///                         `[compatibility]` block.
///   * `artifacts/...`   — copy of every regular file under
///                         `cache_dir`, preserving relative paths.
///
/// Pre-conditions:
///   * `target.kind` must be `Library` or `Program`. Source-kind has
///     no archive (decision 6).
///   * `cache_dir` must exist and be a directory.
///   * `archive_path`'s parent must exist (caller's job).
pub fn stage_archive_with_options(
    target: &DepsManifest,
    arch: TargetArch,
    abi_version: u32,
    cache_dir: &Path,
    archive_path: &Path,
    opts: &StageOptions,
) -> Result<(), String> {
    if matches!(target.kind, ManifestKind::Source) {
        return Err(format!(
            "archive_stage: kind=source is not archived (manifest {:?})",
            target.name
        ));
    }
    if !cache_dir.is_dir() {
        return Err(format!(
            "archive_stage: cache_dir {} not a directory",
            cache_dir.display()
        ));
    }

    let manifest_text = build_archive_manifest_text(target, arch, abi_version, opts)?;

    // Build the tar+zstd in memory; write atomically last.
    let mut tar_bytes: Vec<u8> = Vec::new();
    {
        let mut builder = tar::Builder::new(&mut tar_bytes);
        // 1. manifest.toml at the root.
        let mut header = tar::Header::new_gnu();
        header.set_size(manifest_text.len() as u64);
        header.set_mode(0o644);
        header.set_cksum();
        builder
            .append_data(&mut header, "manifest.toml", manifest_text.as_bytes())
            .map_err(|e| format!("tar append manifest.toml: {e}"))?;

        // 2. artifacts/<every-file-in-cache-dir>.
        let mut files: Vec<PathBuf> = Vec::new();
        collect_files(cache_dir, &mut files)?;
        files.sort();
        for src in &files {
            let rel = src
                .strip_prefix(cache_dir)
                .map_err(|_| format!("strip_prefix {}", src.display()))?
                .to_string_lossy()
                .into_owned();
            let archive_path = format!("artifacts/{rel}");
            let bytes = fs::read(src)
                .map_err(|e| format!("read {}: {e}", src.display()))?;
            let mut h = tar::Header::new_gnu();
            h.set_size(bytes.len() as u64);
            h.set_mode(0o644);
            h.set_cksum();
            builder
                .append_data(&mut h, &archive_path, &bytes[..])
                .map_err(|e| format!("tar append {archive_path}: {e}"))?;
        }
        builder.finish().map_err(|e| format!("tar finish: {e}"))?;
    }

    let mut zst_bytes: Vec<u8> = Vec::new();
    {
        let mut encoder = zstd::stream::write::Encoder::new(&mut zst_bytes, 0)
            .map_err(|e| format!("zstd encoder: {e}"))?;
        encoder.write_all(&tar_bytes).map_err(|e| format!("zstd write: {e}"))?;
        encoder.finish().map_err(|e| format!("zstd finish: {e}"))?;
    }

    // Atomic write: tmp + rename.
    let tmp = archive_path.with_extension("tar.zst.tmp");
    fs::write(&tmp, &zst_bytes)
        .map_err(|e| format!("write {}: {e}", tmp.display()))?;
    fs::rename(&tmp, archive_path)
        .map_err(|e| format!("rename {} -> {}: {e}", tmp.display(), archive_path.display()))?;
    Ok(())
}

fn collect_files(dir: &Path, out: &mut Vec<PathBuf>) -> Result<(), String> {
    for entry in fs::read_dir(dir).map_err(|e| format!("read_dir {}: {e}", dir.display()))? {
        let entry = entry.map_err(|e| format!("read_dir entry: {e}"))?;
        let p = entry.path();
        if p.is_dir() {
            collect_files(&p, out)?;
        } else if p.is_file() {
            out.push(p);
        }
    }
    Ok(())
}

fn build_archive_manifest_text(
    target: &DepsManifest,
    arch: TargetArch,
    abi_version: u32,
    opts: &StageOptions,
) -> Result<String, String> {
    // Read the source deps.toml; append a [compatibility] block. The
    // source deps.toml lives at `<dir>/deps.toml`.
    let src_path = target.dir.join("deps.toml");
    let mut text = fs::read_to_string(&src_path)
        .map_err(|e| format!("read {}: {e}", src_path.display()))?;
    if !text.ends_with('\n') {
        text.push('\n');
    }
    text.push_str(&format!(
        "\n[compatibility]\ntarget_arch = \"{}\"\nabi_versions = [{}]\n\
         cache_key_sha = \"{}\"\nbuild_timestamp = \"{}\"\nbuild_host = \"{}\"\n",
        arch.as_str(),
        abi_version,
        opts.cache_key_sha,
        opts.build_timestamp,
        opts.build_host,
    ));
    // Sanity: this must round-trip through parse_archived.
    let _ = DepsManifest::parse_archived(&text, target.dir.clone())
        .map_err(|e| format!("archived manifest fails its own validator: {e}"))?;
    Ok(text)
}
```

```rust
//! Stage all V2 library + program archives for a release.
//!
//! Walks the registry, fans out across requested arches, calls
//! ensure_built to populate the cache, archive_stage::stage_archive_with_options
//! to produce the .tar.zst, then build_manifest::run to emit
//! manifest.json. Atomic against re-runs.

use std::path::{Path, PathBuf};

pub fn run(args: Vec<String>) -> Result<(), String> {
    // Parse: --staging <dir> [--registry <dir>] [--cache-root <dir>]
    //        [--abi <N>] [--arch wasm32|wasm64]... [--tag <tag>]
    //        [--build-timestamp <iso>] [--build-host <s>]
    //        [--continue-on-error]
    // …

    fs::create_dir_all(staging.join("libs"))?;
    fs::create_dir_all(staging.join("programs"))?;

    let registry = …;
    let mut chain = Vec::new();
    let mut memo = BTreeMap::new();
    let mut errors: Vec<(String, TargetArch, String)> = Vec::new();

    for (_, m) in registry.walk_all()? {
        if !matches!(m.kind, ManifestKind::Library | ManifestKind::Program) {
            continue;
        }
        for &arch in &arches {
            let sha = compute_sha(&m, &registry, arch, abi, &mut memo, &mut chain)?;
            let short = &hex(&sha)[..8];
            let archive_name = format!("{}-{}-rev{}-{}-{}.tar.zst",
                m.name, m.version, m.revision, arch.as_str(), short);
            let subdir = match m.kind {
                ManifestKind::Library => "libs",
                ManifestKind::Program => "programs",
                _ => unreachable!(),
            };
            let archive_path = staging.join(subdir).join(&archive_name);
            if archive_path.exists() {
                eprintln!("skip {} (already staged)", archive_name);
                continue;
            }

            let cache_path = match build_deps::ensure_built(&m, &registry, arch, abi, &resolve_opts) {
                Ok(p) => p,
                Err(e) => {
                    let msg = format!("ensure_built failed: {e}");
                    eprintln!("WARN {} {}: {msg}", m.name, arch.as_str());
                    errors.push((m.name.clone(), arch, msg));
                    continue;
                }
            };
            let opts = StageOptions {
                cache_key_sha: hex(&sha),
                build_timestamp: build_timestamp.clone(),
                build_host: build_host.clone(),
            };
            archive_stage::stage_archive_with_options(
                &m, arch, abi, &cache_path, &archive_path, &opts,
            )?;
            println!("staged {}", archive_path.display());
        }
    }

    // Generate manifest.json by delegating to build_manifest::run.
    build_manifest::run(vec![
        "--in".into(), staging.display().to_string(),
        "--out".into(), staging.join("manifest.json").display().to_string(),
        "--tag".into(), tag.clone(),
        "--abi".into(), abi.to_string(),
        "--registry".into(), registry_root.display().to_string(),
        // pass --arch flags
    ])?;

    // Errors per (manifest, arch) — only fatal if every arch failed.
    if !errors.is_empty() && !continue_on_error {
        let by_manifest: BTreeMap<String, Vec<(TargetArch, String)>> = …;
        let total_failed_per_manifest = arches.len();
        let any_full_failure = by_manifest.values()
            .any(|v| v.len() == total_failed_per_manifest);
        if any_full_failure {
            return Err(format!("stage-release: {} manifest(s) failed every arch — see WARN logs above", by_manifest.len()));
        }
    }
    Ok(())
}
```

```rust
pub fn run(args: Vec<String>) -> Result<(), String> {
    // Parse: --manifest <path> [--archive-base <url-or-dir>]
    //        [--cache-root <dir>] [--local-binaries-dir <dir>]
    //        [--registry <dir>] [--abi <N>]
    // …

    let manifest: Value = serde_json::from_str(&fs::read_to_string(&manifest_path)?)?;
    let entries = manifest["entries"].as_array().ok_or("manifest.entries missing")?;
    let registry = …;
    let abi = …;

    for entry in entries {
        let kind = entry["kind"].as_str().unwrap_or("");
        if !matches!(kind, "library" | "program") { continue; }
        let archive_name = entry["archive_name"].as_str();
        let Some(archive_name) = archive_name else { continue; };  // V1 entries
        let Some(compat) = entry.get("compatibility") else { continue; };

        let program_name = entry["program"].as_str().unwrap();
        let arch = parse_arch(entry["arch"].as_str().unwrap())?;

        // Look up the source manifest from the registry.
        let m = registry.load(program_name)?;

        // Compute archive URL. Either embedded in entry, or derived from --archive-base.
        let archive_url = compute_archive_url(entry, &archive_base, kind, archive_name)?;
        let archive_sha = entry["archive_sha256"].as_str().unwrap().to_string();
        let bin = Binary { archive_url, archive_sha256: archive_sha };

        // Local cache_key_sha — must agree with archive's compat.cache_key_sha.
        let mut chain = Vec::new();
        let mut memo = BTreeMap::new();
        let local_sha_bytes = compute_sha(&m, &registry, arch, abi, &mut memo, &mut chain)?;
        let local_sha = hex(&local_sha_bytes);

        let canonical = canonical_path(&cache_root, &m, arch, &local_sha_bytes);

        if canonical.exists() {
            eprintln!("skip {} (already in cache)", canonical.display());
        } else {
            remote_fetch::fetch_and_install(&bin, &canonical, &m, arch, abi, &local_sha)
                .map_err(|e| format!("{}/{}: {e}", program_name, arch.as_str()))?;
            println!("installed {}", canonical.display());
        }

        if matches!(m.kind, ManifestKind::Program) {
            mirror_outputs_to_local_binaries(&m, &canonical, &local_binaries_dir)?;
        }
    }
    Ok(())
}

fn mirror_outputs_to_local_binaries(
    m: &DepsManifest, canonical: &Path, local_binaries_dir: &Path,
) -> Result<(), String> {
    let outputs = &m.program_outputs;
    let dest_dir = if outputs.len() > 1 {
        local_binaries_dir.join("programs").join(&m.name)
    } else {
        local_binaries_dir.join("programs")
    };
    fs::create_dir_all(&dest_dir)?;
    for out in outputs {
        let src = canonical.join(&out.wasm);
        let dest_name = if outputs.len() > 1 {
            format!("{}.wasm", out.name)
        } else {
            format!("{}.wasm", out.name)
        };
        let dest = dest_dir.join(&dest_name);
        // Atomic copy: write tmp + rename.
        let tmp = dest.with_extension("wasm.tmp");
        fs::copy(&src, &tmp)?;
        fs::rename(&tmp, &dest)?;
    }
    Ok(())
}
```
