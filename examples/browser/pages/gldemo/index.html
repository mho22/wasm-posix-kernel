Spinning Cube — fork(2) + pipe(2) + GLES2

A single C source file (programs/cube.c) that fork()s into two cooperating processes: the child rotates the cube in 3D and projects to clip space; the parent renders the resulting triangles via the kernel's /dev/dri/renderD128 WebGL2 bridge. They communicate through a real pipe(2) — no JS bootstrap per process, no MessagePort, just kernel pipe buffers.

Click Start to spawn the cube.

The child writes one frame's worth of transformed vertices (36 triangle verts × 6 floats = 864 bytes) per pipe write — below PIPE_BUF, so each frame is delivered atomically. FPS lines on the right come from the parent's printf over the kernel's stdout — proof the POSIX side is alive.
